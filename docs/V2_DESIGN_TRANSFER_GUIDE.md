## §1. なぜ V2 を記憶荘にも適用するのか

満室で確立された**「楔＋堀＋Poacher-turned-gamekeeper」の物語構造**は、記憶荘でも通用する。理由:

- **同じ 2人チーム**（KAI + KEN）= 堀が共通
- **同じ 20年 × 20年 = 40年の現場**
- **KAIの元仲介→職人側への転身物語**は業態（店舗 vs アパート）に依存しない
- **楔（テナント探しから手伝う内装屋）は CPA¥1,362 で実証済み**、方向性は正しい

ただし満室とは違い、**記憶荘 LP は広告実証済みなので慎重に扱う必要がある**（§4 参照）。

## §3. 推奨着手順序

満室とは違い、**記憶荘 LP は CPA¥1,362 で回っている**。闇雲に触るとロスになる。

### 着手順序（リスクの低い順）

1. **[最初]** 現状把握
   - 現 HP `/` を読む
   - 現 LP `/lp/tenant` を読む
   - `CLAUDE.md` と `memory/MEMORY.md` 経由で最新状況確認
   - `git log --oneline -10` で最新コミット確認

2. **[Phase 1] HP Hero の V2 化**
   - 広告直撃 LP は触らない
   - ブランドページの HP のみ先行
   - V2 プレイブックの パターンA を適用
   - ターゲットバッジ: 「宮城で店舗を出す方向け（飲食・美容・サロン）」
   - H1 候補: 「テナントを紹介する側にいた人間が、裏側を教えます。」
   - それ以外の Hero 要素（顔写真・数字プラーク・40年行・✓ピル）は満室とほぼ共通

3. **[Phase 2] HP 他セクションの V2 トーン化**
   - `SectionHeading` を満室版と同じ両翼ヘアラインに
   - `Voice` 系に金リング顔写真
   - 料金バッジを◆コーナー + Cormorant 数字のプラーク化

4. **[Phase 3] About ページの V2 化**
   - Hero: 140px 金リング顔写真 + 数字プラーク
   - KAI's Story + KEN's Story セクション
   - Business Model 比較図（店舗内装版: お客→仲介→元請け→下請け vs お客→KAI×KEN）
   - Philosophy / Approach

5. **[Phase 4] LP `/lp/tenant` の V2 化 ← 最後**
   - **必ず A/B テスト形態で**（既存LPを上書きしない）
   - `/mock/tenant-v2` のような別ルートで並走
   - 広告側の一部配分で実データ取得
   - 満室の BukkenHero V2 の店舗版を作る
   - 「物件」→「テナント」置換で構造を維持

## §4. 既存の価値を壊さない原則（絶対厳守）

### 広告・LP 関連

- **CPA¥1,362 の実績がある LP を無検証で上書き禁止**
- 変更は必ず A/B テスト形態で（新ルート作成 → 広告側で一部配分）
- 楔（H1・広告見出し）を変える場合は広告側と連動必須
- 「非公開物件」のような**記憶荘の既存概念**を勝手に変更しない

### デプロイ関連（過去の事故を踏まえて）

- **`_redirects` の `!` 禁止**（自己リダイレクトループを起こす）
- デプロイ後は **必ず curl で疎通確認**
- Netlify site 紐付けを毎回確認（`manshitsu-homes` と `hp-lp` は別サイト）
- **broken state を検知する Clarity 0件監視**を意識

### コンテンツ関連

- `CLAUDE.md` の **LP運用ルール**（「一緒に勝つ」等禁止ワード）は V2 でも適用
- 「店舗内装 費用」「内装工事 費用」系の費用起点訴求は使わない（実顧客化しないと判明）
- 「断ってOK」「営業なし」「相見積もり歓迎」を全LPで明示する
- 記憶荘は **飲食・美容・サロン中心**が顧客層、これは変えない

## §5. V2 適用の具体的差分（店舗 vs アパート）

| 要素 | 満室（アパート） | 記憶荘（店舗内装） |
|---|---|---|
| ターゲットバッジ | 宮城の収益物件オーナー向け | **宮城で店舗を出す方向け（飲食・美容・サロン）** |
| 職歴ラベル | 元不動産仲介20年 × 大工20年 | **同文**（共通） |
| 顔写真 | KAI / KEN | **同じ**（about-kai-sm.webp / about-ken-sm.webp） |
| H1 候補 | 物件を売る側にいた人間が、裏側を教えます | **テナントを紹介する側にいた人間が、裏側を教えます** |
| サブ | 20年やって、辞めました | **同文** |
| 数字プラーク | 20年 / 20年+ / 30件+/年 | **同じ**（KEN の年間施工は店舗内装が主軸なので整合） |
| 40年行 | 2人で40年の現場 | **同文** |
| コミットピル | 月5組限定 / 下請けに出さず / 売買で儲けない / 営業電話なし / 相見積OK | **同じ**（or 「断ってOK」等を追加） |
| 物件ちら見せ | 非公開収益物件リスト | **非公開テナントリスト**（既存要素を転用） |
| パターン・インタラプト | 公開物件は残り物 / 利回りの裏 / 修繕コストのズレ | **公開テナントは残り物 / 家賃だけで選ぶ危険 / 内装費見積りのズレ** |
| 三角形 | 元仲介 / 職人 / 中立 | **同じ** |
| About Story | 元仲介20年、辞めた理由 | **同じ**（同じKAI） |
| Business Model 比較図 | 一般 4段 vs 満室 1社 | **同じ構造**（店舗内装でも中抜き構造は同じ） |

### 広告 LP で特に気をつけること

- 既存の「テナント探しから手伝う内装屋」**コンセプトは維持**
- V2 適用時、H1 を変えるなら広告の**見出しテキストも連動修正**
- **Clarity で既存 LP と新 LP の行動差を監視**（離脱率・スクロール深度）
